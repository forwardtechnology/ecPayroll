Change Employee ID

Make sure you have completed following session(s)

l   Add New Employee

 

 

 

You are NOT suggested to change the Employee ID unless necessary.

 

In a Human Resources management System, ¡§Employee ID¡¨ is the identifier of an employee. It should be unique, simple, meaningless, and not be re-used. Simple running number (e.g. 0001, 0002 ¡K) may be a good approach to assigning Employee ID. If there are some meanings embedded in the Employee ID, for example ¡§A180001¡¨ where ¡§A¡¨ means ¡§Department A¡¨ while ¡§18¡¨ means employed in 2018, you may not wish to see if the staff was moved to another department, or the staff was re-employed in another year because that means a new Employee ID has to be assigned to this staff. If a staff owns more than 1 Employee ID, it is difficult to manage, search, and viewing history. If you are re-using the Employee ID to assign to multi persons even not at the same time, it can be a nightmare.

 

If this really happen, you can still change the Employee ID in ecPayroll system but it is not suggested.

 

Steps

1.          In main screen, double-click the employee, and go into the employee page.

 

 

 

2.          Click the button [Edit] next to the field ¡§Staff code¡¨ (or ¡§Employee ID¡¨), to open ¡§Employee ID history¡¨ screen.

 

 

3.          Press the button [Change ID], to open another screen ¡§Employee ID Update¡¨.

 

 

4.          Enter a new Employee ID, and some remarks saying why you need to do this. Press [OK] button to confirm.

 

 

5.          Now the Employee ID was changed, and the previous IDs was recorded.

 

 

6.          Finished. From now on, to search or filtering this staff, you should use the new Employee ID. However, you can still do the search with the old Employee ID by using the EE Filter > Previous employee ID
