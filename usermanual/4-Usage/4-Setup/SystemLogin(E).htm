System Login (Administrator)

Make sure you have completed following session(s)

l   Login User Setup

 

Every time when you enter the system (i.e. TAMS Windows client), you are required to input your own user account name and password. System will check your password, identify who you are, checking what you can do, and record all your important actions.

 

Steps

1.          Enter your assigned USERNAME

 

 

2.          Enter corresponding PASSWORD

 

 

3.          Press the [OK] button to login to the system

 

 

4.          If login success, the system main screen will be shown as below

 

 

 

NOTE

If login fail, please check all information are corrected. Below are some common reasons that cause login failed :

 

Cause

Description

Solution

Username not exist

User account must be created before login.

Create User account in “User Setup” by System administrator

Incorrect Username

Username not matched.

Note that it is case insensitive.

No space is allowed.

1.      Retry, or

2.      Checked by System administrator

Incorrect Password

Password not matched.

Note that it is case insensitive.

No space is allowed.

1.      Retry, or

2.      Reset password by System administrator

Too many users logged-in to the system

Total number of logged-in users may already reached the “Maximum concurrent user license” purchased.

1.      Retry after another user logout the system, or

2.      If no any other user is using the system, request system administrator to :

            i.                force logout another un-used user license, or

          ii.                reset the dongle, or

3.      Purchase more user licenses.
