DateLock Protection

Make sure you have completed following session(s)

l   None

 

DateLock is a data protection function to prevent un-authorised data amendment or deletion. All records of selected data type which before the cut-off date will be protected. There are 5 data types can be protected by using the DateLock function

1.        Duty Roster

2.        Leave Records

3.        Time attendance records

4.        Payment records

5.        Salary records

 

Steps

1.            Click [File] > [DateLock Protection] in Main Screen pull down menu.

 

2.            Choose a category to apply the DateLock. E.g. [Duty Roster]

 

3.            Click the check box to enable the lock protection

 

4.            Choose a lock mode ( [Manual] or [Automatic] )

i.e.     [Manual] > Input a cut-off date to control the lock period. All data before cut-off date will be protected.

[Automatic] > System auto lock each month data by checking user pre-defined cut-off date of each month

 

5.            For [Manual] Mode, Input the cut-off date for data protection. For below example, input ¡§2015/01/01¡¨ means all previous data of Duty Roster which before 2015/01/01 are locked. Data cannot be modified or deleted.

 

6.            For [Automatic] mode, input cut-off date for each month. In below example, set [Feb 02] as the cut-off date for January, means the Duty Roster in January (i.e. 1/1-1/31) will be locked after Feb 02. Data cannot be modified or deleted.

 

7.            Press [Update] button to save above configuration

 

8.            Repeat the steps 2 ¡V 7 if you want to enable lock for other data type
