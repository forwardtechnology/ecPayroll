當登入ecPayroll系統時出現如下錯誤，則需要檢查硬體鎖

 

檢查步驟:

1.          在伺服器PC(硬體鎖所在的PC)檢查[R]圖示是否正在執行(如下圖)

**註：如硬體鎖安裝在window vista / window 7 平台上, 沒有[R]圖示屬正常。請於 控制台 -> 系統管理工具 -> 服務 查看NetDogService 查看此服務是否正在執行

 

2.          如沒有[R]圖示或NetDogService狀態為[已停用]，請於程式集->NetDog Development Kits V2.3 -> NetDog Server -> Start NetDog Server 啟動服務

 

3.          如[R]圖示正在執行，請右click圖示按Open Server Management

 

4.          選取[ecPayroll]，查看當前的[使用者數量]和[最大使用者數量] (如當前使用者數量與最大使用者數量相同，新的使用者將不能登入ecPayroll，請關閉部份ecPayroll的使用者以釋放使用者數量)

 

5.          如[ecPayroll]圖示出現[紅色底色]，即表示電腦找不到硬件鎖。請檢查USB硬件鎖是否牢固的插在電腦上，請嘗試將USB硬件鎖插在電腦的其他USB埠上。並按[File -> Refresh]查看[ecPayroll]圖示有否回復正常

       

 

6.          如上述步驟未能解決硬體鎖問題，請嘗試重新啟動硬體鎖服務 ( 控制台 -> 系統管理工具 -> NetDogService 右click 停止, 然後按 啟動. )

 

7.          如重新啟動服務未能解決硬體鎖問題，可能是該服務被防火牆攔截或網絡狀態不穩定，請根據以下方法處理：

 

7.1      設定防火牆方法:

開啟 控制器 -> windows 防火牆 -> 例外

 

按 新增程式 瀏覽 C:\Windows\system32\NetDogSrv.exe 然後按 確定

 

7.2      設定於固定PC檢查硬體鎖的方法 (安裝硬體鎖PC必須使用固定IP)

於 伺服器 -> TAMS -> USBDongleSetup 開啟 gsnetlog.ini

修改ND_SERVER_ADDR 的IP (請塡上安裝硬體鎖PC的IP, 只接受IP輸入請勿塡寫電腦名稱)

儲存gsnetlog.ini，並將gsnetlog.ini放在 伺服器\TAMS\ 內
